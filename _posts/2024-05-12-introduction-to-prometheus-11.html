Selectors and Matchers

In Prometheus Server, we may have hundreds of different time series with different label values. But we only want to keep track of one time serie. In this case, we need to narrow down the time series and only look at the time series we want to work on. This is when Matchers come into place. For example, we get 2 outputs from process_cpu_seconds_total. But we just want the output with job="node_exporter".

process_cpu_seconds_total{job="node_exporter"}

There are some Matcher types

• Equality matchers (=) - Select labels that are exactly equal to the provided string 

       Ex: process_cpu_seconds_total{job='node_exporter'}
• Negative Equality matcher (!=) - Select labels that are not equal to the provided string. 
       Ex: process_cpu_seconds_total{job!='node_exporter'}
• Regular expression matcher (=~) - Select labels that regex-match with the provided string. 
       Ex: prometheus_http_requests_total{handler=~"/api.*"}
• Negative Regular expression matcher (!~) - Select labels that do not regex-match with the provided string. 
       Ex: prometheus_http_requests_total{handler!~"/api.*"}

These Matchers are pretty easy to use and really helpful. Hope you have fun with it.

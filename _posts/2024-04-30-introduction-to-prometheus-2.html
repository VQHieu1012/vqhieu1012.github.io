Basic Terminologies in Prometheus

Monitoring

• Monitoring is a systematic process of collecting and recording the activities taking place in a target project, program or service and then using that recorded values to check if the targets are reaching their objectives or not.

Alert/Alerting

• An alert is the outcome of an alerting rule in Prometheus that is actively firing. Alerts are sent from Prometheus to the Alertmanager.

Alertmanager

• The Alertmanager takes in alerts from Prometheus server, aggregates them into groups, de-duplicates, applies silences, throttles, and then sends out notifications to email, Pagerduty, Slack, etc.

Target

• A target is the definition of an object to scrape. Target is an object whose metrics are to be monitored. For example, target can be a Linux machine, Windows machine or your own set of applications.

Instance

• In Prometheus terms, an endpoint you can scrape is called an instance.
Ex: 1.2.3.4:5678, 1.2.3.4:5679,...

Job

• Job is a collection of targets/instances with the same purpose.
For example, an API server job with 4 replicated intances: → job: api-server • instance 1: 1.2.3.4:5670 • instance 2: 1.2.3.4:5671 • instance 3: 1.2.3.4:5672 • instance 4: 1.2.3.4:5673 We consider each address is one instance. Grouping all these addresses together comes as one job.

Sample

• A sample is a single value at a point in time in a time series.

• Ex: http_requests_total{method="get"} → 32

Alright. So these are some basic and widely used terminologies in Prometheus. We will encounter some new terminologies but we will expain it as we go along.

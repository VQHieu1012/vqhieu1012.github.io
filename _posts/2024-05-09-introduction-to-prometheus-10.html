Querying Basics

Using PromQL not only offers user to select from Time series, but also offers the ability to do all sorts of aggregations, analyse and do automatic operations just like any other relational database query. This allows user to have better control of their stored metric data, which eventually helps in better understanding the performance of their system metrics. Before we start getting in PromQL, let's first learn about data types that PromQL expression or sub-expression can evaluate.

Data Types

There are 4 data types that PromQL expression evaluates: Instant Vector, Range Vector, Scalar, String.

• Instant Vector - A set of time series containing a single sample for each time series, all sharing the same timestamp (one sample for each time series).

prometheus_http_requests_total

• Range Vector - A set of time series containing a range of data points over time for each time series (many samples for each time series).

prometheus_http_requests_total[2m]

• Scalar - A simple numeric floating point value.

   Ex:  12.34

• String - A simple string value, currently unused in Prometheus.

   Ex:   hello world

And these are data types that Prometheus can evaluate.

Introduction to Prometheus

In the first post about Introduction to Prometheus, we will lay the foundation of our learnings by understanding what is Prometheus, starting with its definition. According to its official documentation:

Prometheus is an open-source systems monitoring and alerting toolkit with an active ecosystem. It is an open-source, metric-based monitoring system that collects metrics from the targets being monitored.

→ It's a tool that allows you to analyze how your applications and infrastructures are performing from the metrics discovered by it.

→ Most Prometheus components are written in Go, which makes them easy to build and deploy as static binaries.

→ It uses a multi-dimentional time series database where every time series is uniquely identified by its 'metric name' and 'key/value pairs' that are generally called labels.

Get total of http requests

→ To fetch the data from its Time series data model, Prometheus uses simple and powerful language known as PromQL (Prometheus Query Language). It is a 'read-only' and flexible query language, that allows aggregation across any of the labels stored in its time series.

→ Since distributed systems are challenging to make reliable, Prometheus does not attemp to form any clusters. It is built to work on a single server node and it stores metrics in memory and local disk. In short, Prometheus does not rely on distributed storage.

→ To monitor custom services, you can add instrumentation to your code, via Prometheus client libraries like Go, Java, Python and many more.

→ Full-fledged monitoring system with its own Alertmanager.

How about talking a little non-technical about Prometheus project?

Prometheus was started in 2012 at SoundCloud. And later on, it joined the Cloud Native Computing Foundation. In 2016, it became the second hosted project of Cloud Native after Kubernetes. Prometheus alse has very active developer and user community around the globe.

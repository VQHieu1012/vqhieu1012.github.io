In this post, we are going to walk through the UI again and know how and where Prometheus scrapes data from its targets.

First, we need to run Prometheus service.

On starting Prometheus, it first goes to the configuration file (prometheus.yml) and loads the settings from that file. As we saw in previous post, those are settings like what to scrape, where to scrape and all other settings. Base on configurations, it can run up Prometheus instance. And now let's head to the UI.

In the Expression box, type "up" and execute. "up" command returns a list of targets scraped by Prometheus. At this time, we only have 1 job defined in Prometheus configuration file.

Let's access the endpoint where Prometheus actually scrapes these data. We go to Status -> Target.

If we redirect to the endpoint http://localhost:9090/metrics, this is where Prometheus scrapes the data. It is a long list of data that Prometheus exposes about itself at this endpoint.

As we have seen in the configuration file, the default value for scrape_interval is 15s, Prometheus will access to the endpoint every 15s to get the sample value for the metrics and store those values in a time series database. From there, every UI tool can access to database and visualize the time series data.

And that is how Prometheus collects data from its targets.

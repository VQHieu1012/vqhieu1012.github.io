Installation

At first, we will need to install Prometheus. There are some ways to install it. You can take a look at the document. After installation, start up your Prometheus. By default, you can access to Prometheus UI in localhost with port 9090.

Graph

At first look under the page, we can see an expression input box from where we execute our queries. This expression browser (aka expression input box) is useful for running ad hoc queries, developing from expressions and debugging both PromQL and the data inside Prometheus.

Let's try with a simple expression "up" and press "Execute".

By clicking on "Execute", we will execute a specified expression and will return a Time series with metric name "up". The "up" metric is a special metric that outputs the list of targets scraped by Prometheus. By default, Prometheus also scrapes itself and exposes data about itself as a HTTP endpoint as we see in the above image. The instance label indicates the scraped target, which is at localhost:9090 (the port Prometheus running) and job (with the value "Prometheus" indicates the type of application). For the "up" expression, the value in second column displays the sample value collected at that time. The value "1" (in this "up") indicates that the script was successful.

Moving ahead, we have a box "Evaluation time". Since Prometheus scrapes data from targets after the specified time interval, for example 15 seconds or 1 minute, we can filter out the result for a specific date time.

The next thing we will use is Graph view.

In this view, we can see the graph of our metrics to know how they change over time. For the "up" metric, the value does not change over time, so we get a straight line.

Let's try another metric: prometheus_http_requests_total

This is the time series data has stored in its database for this metric. We visualize it using stacked graph.

Alerts

From this page, we can get the list of all alerts we have configured with their alert status. This page is empty because we haven't defined any alert. We can define them in Alertmanager.

Status Navigation Bar

Runtime & Build information

This page displays the genenal information (working directory, Configuration reload, Version...).

Command-Line Flags

We have some command line settings.

Configuration

Configuration page displays all the running Prometheus settings. These settings are coming from Configuration file.

Rules

Rules page displays the rules Configuration we have defined in our rules file.

Targets

Targets page shows all target nodes, which Prometheus scrapes data from.

Service Discovery

This page shows a list of discovered targets with their corresponding labels

Help

This bar just redirect you to the Prometheus official website.

So that is all for Prometheus UI. We recommend you to interact with the UI yourself to get familiar to it.

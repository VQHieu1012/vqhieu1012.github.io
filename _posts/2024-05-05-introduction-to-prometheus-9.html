In this post, we will see how to get Windows machine metrics using WMI exporter (Windows Management Instrumentation). WMI exporter is not available in Prometheus official documentatin. So we need to download it externally. We can download WMI exporter in this Github repository

In our laptop, we use windows_exporter-0.25.1-amd64.exe released on Jan 17.

After we have downloaded the WMI, we can run easily run it with just one click.

WMI exporter is now running on localhost:9182. Let's head to the UI and access it.

If we access to metrics, we will get a list of infomation our Windows system has exposed.

WMI is ready. The next step is connect our Windows system with Prometheus Server. As we ran WMI on Windows machine and Prometheus Server on VMWare, we have to make sure the Prometheus Server can communicate with Windows machine. To get our local IP address, we can run ipconfig in Windows terminal.

And we need to install openssh-client and openssh-server on Prometheus server. We can try ping to Windows IP address to make sure they can communicate with each other.

Or we can use browser and access to the IP and port 9182 in Prometheus Server. In our case is 192.168.23.1:9182
